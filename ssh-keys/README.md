# SSH-ключи

## Что такое SSH-ключи

SSH-ключи — это способ безопасно подтверждать свою личность при подключении к серверу или сервису, например GitHub, без ввода пароля от аккаунта.

SSH-ключ состоит из пары:

- private key — приватный ключ;
- public key — публичный ключ.

Приватный ключ хранится только на компьютере пользователя и никому не передаётся. Публичный ключ можно добавлять на серверы, GitHub, GitLab и другие сервисы.

## Зачем SSH-ключи нужны вместо пароля

SSH-ключи удобнее и безопаснее пароля, потому что приватный ключ не отправляется на сервер при подключении.

Сервер хранит публичный ключ. Компьютер пользователя доказывает, что у него есть подходящий приватный ключ. Если проверка проходит успешно, подключение разрешается.

Преимущества SSH-ключей:

- не нужно вводить пароль от аккаунта при каждом подключении;
- приватный ключ остаётся на компьютере пользователя;
- ключ можно дополнительно защитить passphrase;
- доступ можно быстро отозвать, удалив публичный ключ на сервере или в GitHub.

## Как выпустить SSH-ключ

Для генерации ключа использовалась команда:

```bash
ssh-keygen -t ed25519 -C "ssh-keys-task"
```

Что означает команда:

- `ssh-keygen` — утилита для создания SSH-ключей;
- `-t ed25519` — тип создаваемого ключа;
- `-C "ssh-keys-task"` — комментарий к ключу, чтобы было понятно, для чего он создан.

Во время генерации был указан отдельный путь для ключа:

```bash
/Users/pln.kmb/.ssh/ssh_keys_task_ed25519
```

Это нужно, чтобы не перезаписать основной SSH-ключ.

## Какие файлы появились после генерации

После генерации были созданы два файла:

```bash
/Users/pln.kmb/.ssh/ssh_keys_task_ed25519
/Users/pln.kmb/.ssh/ssh_keys_task_ed25519.pub
```

Первый файл без `.pub` — это приватный ключ. Его нельзя никому отправлять, публиковать или добавлять в GitHub.

Второй файл с `.pub` — это публичный ключ. Его можно добавлять на серверы, GitHub, GitLab и другие сервисы.

Проверка созданных файлов:

```bash
ls -la ~/.ssh | grep ssh_keys_task
```

Результат:

```bash
-rw-------@  1 pln.kmb  staff  444 Apr 24 14:14 ssh_keys_task_ed25519
-rw-r--r--@  1 pln.kmb  staff   95 Apr 24 14:14 ssh_keys_task_ed25519.pub
```

## Куда добавляется публичный ключ

Публичный ключ добавляется туда, куда пользователь хочет подключаться по SSH.

Например, для GitHub публичный ключ добавляется в настройках аккаунта:

```text
GitHub → Settings → SSH and GPG keys → New SSH key
```

Для этого задания был добавлен публичный ключ:

```bash
cat ~/.ssh/ssh_keys_task_ed25519.pub
```

Публичный ключ начинался с:

```text
ssh-ed25519
```

После добавления публичного ключа GitHub может узнавать пользователя по соответствующему приватному ключу.

## Как использовать SSH-ключ

Чтобы SSH мог использовать ключ, приватный ключ был добавлен в `ssh-agent`:

```bash
ssh-add ~/.ssh/ssh_keys_task_ed25519
```

Что означает команда:

- `ssh-add` — добавляет приватный ключ в `ssh-agent`;
- `~/.ssh/ssh_keys_task_ed25519` — путь к приватному ключу;
- файл указывается без `.pub`, потому что для подключения используется приватная часть.

Проверка, что ключ добавлен в `ssh-agent`:

```bash
ssh-add -l
```

Результат:

```bash
256 SHA256:Z+6XvhoZ2BNgT/hFuCjkBOGJKWXSEvkqQhWvwaNbaUY ssh-keys-task (ED25519)
```

## Проверка подключения по SSH

До добавления публичного ключа в GitHub проверка завершалась ошибкой:

```bash
ssh -T -i ~/.ssh/ssh_keys_task_ed25519 git@github.com
```

Результат:

```bash
git@github.com: Permission denied (publickey).
```

Это произошло потому, что GitHub ещё не знал публичную часть ключа.

После добавления публичного ключа в GitHub проверка прошла успешно:

```bash
ssh -T -i ~/.ssh/ssh_keys_task_ed25519 git@github.com
```

Результат:

```bash
Hi plnkmb! You've successfully authenticated, but GitHub does not provide shell access.
```

Это означает, что GitHub узнал ключ и SSH-аутентификация работает.

## Основные типы SSH-ключей

На практике чаще всего встречаются такие типы SSH-ключей:

### Ed25519

Современный рекомендуемый тип ключа. Он короткий, быстрый и считается безопасным для обычного использования.

Пример генерации:

```bash
ssh-keygen -t ed25519
```

### RSA

Старый и очень распространённый тип ключа. Всё ещё встречается часто, особенно в старых системах.

Для RSA важно использовать достаточную длину ключа, например 4096 бит:

```bash
ssh-keygen -t rsa -b 4096
```

### ECDSA

Тоже современный тип ключа, но на практике его используют реже, чем Ed25519 и RSA.

Обычно для новых ключей лучше выбирать Ed25519, если нет особых требований от старой системы.

## Базовые правила безопасности

Главное правило: приватный ключ нельзя никому передавать.

Что важно соблюдать:

- не отправлять приватный ключ в чаты, почту, Jira, GitHub или другие публичные места;
- добавлять на серверы и в GitHub только публичный ключ с расширением `.pub`;
- защищать приватный ключ `passphrase`;
- не хранить приватный ключ в репозитории;
- если приватный ключ был случайно опубликован, его нужно считать скомпрометированным и создать новый;
- удалять публичный ключ из GitHub или сервера, если доступ больше не нужен.

Приватный ключ — это как ключ от квартиры. Публичный ключ — это как замок, который можно поставить на дверь. Замок можно показать, ключ от замка — нельзя.
